
step 1 : create a new folder for playwright project  and open in vs code 

step 2 : create a python virtual env for this project using below command 
	 
	 python -m venv my_env 

step 3 : to activate the env use below

	my_env\scripts\activate

step 4 : once env active then inside run below command 

	pip install playwright (this is useful for python to talk to browser)

(optional) to upgrade playwright use this command -->pip install playwright -u

step 5 : now we need to install actual browsers(Chromium, Firefox, and WebKit)
	 that can be used by python to run test runner for that run below command 
	 
         playwright install

step 6 : verify installation by running below command 

	python -m playwright --version or playwright --version 	  

step 7 : run below command to install pytest plugin for playwright and testing framework in python

	 pip install pytest-playwright ...(if you directly run this command you can skip 4)

         .....(you can use this command pip list to list down all packages downloaded and 
	      also if you use this command pip show pytest it will give you detailed description of it)

step 8 : now the setup is ready , now create a folder inside your playwright project give name tests

step 9 : create a python file name test_google.py ,inside this file write code on how to 

	 open google and write anything on search bar and click search (code already created using chat gpt please refer test_google.py inside Plywright_learning folder)

step 10: after the code is successfully created and run 

	 run this command pytest --help   (it will help you  give tags that can be run along, when running the code from terminal ..for eg: pytest tests\test_google.py --browser=chromium)

step 11: create folders inside root folder (playwright learning) utils, pages, reports and tests..(ignore tests if already created in step 8)

step 12: create conftest.py file under root folder (playwright learning)

         (about conftest.py imagine you are testing 10 different websites 

         without conftest.py you would need to write this same code in every test file 

         this repeating code is not good practice

         with conftest.py we define that once as a fixture and just use it in tests)

step 13: then watch this video to understand pom 

	 https://www.youtube.com/watch?v=h-fpoNZdWCg&t=1666s
